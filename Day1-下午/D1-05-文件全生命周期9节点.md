## 一、文件全生命周期 9 节点

```
① 编制 → ② 审核 → ③ 批准 → ④ 编号 → ⑤ 发放
          ↓
⑨ 销毁 ← ⑧ 作废 ← ⑦ 回收 ← ⑥ 使用
```

---

## 二、9 节点详解（必背）

### ① 编制（Drafting）
- **责任人**：文件起草人（工程师/主管/文员）
- **输入**：业务需求、标准更新、法规变化
- **输出**：文件草稿（Word/Excel）
- **规范**：套用公司模板、有页眉页脚、有版本号栏、有受控标识栏
- **面试话术**："起草时用公司统一的 SOP 模板，确保格式统一"

### ② 审核（Review）
- **责任人**：相关业务部门负责人
- **输入**：文件草稿
- **输出**：审核意见（书面或在文件上批注）
- **关注点**：业务匹配性、可操作性、与上下层文件的一致性
- **技巧**：跨部门文件需多部门会签

### ③ 批准（Approval）
- **责任人**：体系经理 / 部门经理 / 总经理（看层级）
- **输出**：签字 + 日期
- **要点**：批准后才能生效
- **面试话术**："所有文件必须经授权人书面批准，未经批准的文件不发放"

### ④ 编号（Numbering）
- **责任人**：**文控岗**
- **规则**：按公司《文件管理办法》的编号规则
- **输出**：唯一文件编号（如 QA-SOP-005-V1.0）
- **要点**：
  - 编号与版本号同时确定
  - 一旦发放，编号永不复用
  - 作废文件编号占号，新文件用新号

### ⑤ 发放（Distribution）
- **责任人**：文控岗
- **方式**：
  - **纸质**：原件留档 + 受控副本盖蓝色受控章 + 发放登记
  - **电子件**：上传到 ERP/PLM/DCC 系统 + 邮件通知 + 权限设置
- **登记本**（必备）：
  | 日期 | 文件编号 | 文件名 | 版本 | 发放对象 | 接收人签字 |
- **要点**：必须**收回签收单**才算完成发放

### ⑥ 使用（In Use）
- **责任人**：现场操作人员
- **要求**：
  - 现场使用的文件必须**最新版**
  - 现场必须有**受控标识**（受控章/受控标签）
  - 现场不得出现**手写修改**（如要改必须走 ECN）
- **文控岗巡检**（每周 1 次）：抽查现场文件版本

### ⑦ 回收（Retrieval）
- **触发条件**：
  - 文件更新版本（V1.0 → V2.0）
  - 文件作废
  - 部门/人员调动（离开该岗位）
- **责任人**：文控岗
- **动作**：
  - 发《文件回收通知》给持有人
  - 收回旧版文件
  - 在《文件发放回收登记表》上勾掉
  - 旧版文件贴"作废"红章

### ⑧ 作废（Obsolescence）
- **责任人**：文控岗
- **动作**：
  - 在受控清单上标"作废"
  - 旧版文件统一保管（防止误用）
  - 保留期：质量记录 ≥ 3 年（部分 ≥ 产品生命周期 + 1 年）

### ⑨ 销毁（Destruction）
- **触发条件**：保留期满
- **责任人**：文控岗 + 部门主管
- **方式**：
  - 纸质：碎纸机 / 焚烧（要记录）
  - 电子：彻底删除 + 备份清除
- **记录**：《文件销毁记录》——日期、文件号、数量、销毁方式、监销人

## 三、文件全流程图（画出来贴墙上）

```
业务需求 → 起草 → 审核 → 批准 → [文控岗介入]
                              ↓
                          编号 + 受控
                              ↓
                          发放（纸质/电子）
                              ↓
                          现场使用 ←── 文控岗周巡检
                              ↓
                          更新/废止触发
                              ↓
                          回收旧版
                              ↓
                          作废登记
                              ↓
                          保留期（≥3年）
                              ↓
                          销毁 + 记录
```

## 四、面试杀手锏：9 节点表 + 9 张表单

> 面试 Q：你能描述一下一个文件从起草到销毁的全过程吗？

**答**：
> 一个文件要经历 9 个阶段：编制、审核、批准、编号、发放、使用、回收、作废、销毁。文控岗介入最深的环节是**编号、发放、回收、作废、销毁**这 5 个。我会维护 9 张表单：编制任务单、审核意见单、批准记录、文件总清单、发放回收登记表、现场文件巡检表、作废登记表、销毁审批单、销毁记录。

| 节点 | 文控岗动作 | 必备表单 |
|---|---|---|
| ① 编制 | 提供模板、编号预留 | 《文件编制任务单》 |
| ② 审核 | 收审意见 | 《审核意见单》 |
| ③ 批准 | 收签字 | 《文件批准记录》 |
| ④ **编号** | **分配唯一编号** | **《文件编号申请表》** |
| ⑤ **发放** | **盖章 + 登记 + 通知** | **《文件发放回收登记表》** |
| ⑥ 使用 | 周巡检 | 《现场文件巡检表》 |
| ⑦ **回收** | **追回旧版** | **《文件回收通知》** |
| ⑧ 作废 | 红章 + 隔离 | 《作废文件登记表》 |
| ⑨ **销毁** | **审批 + 监销** | **《文件销毁审批单》+ 《销毁记录》** |

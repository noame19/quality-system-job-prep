## 一、ISO 9001 三大支柱（**必背**）

ISO 9001:2015 引言 0.3 明确：**过程方法 + PDCA + 基于风险的思维** 是标准的三块基石。

```
          ┌──────────────────────────┐
          │    ISO 9001 三大支柱     │
          └──────────┬───────────────┘
                     │
       ┌─────────────┼─────────────┐
       │             │             │
       ▼             ▼             ▼
   过程方法        PDCA        基于风险的
  (Process       (Plan-Do-     思维 (Risk-
  Approach)      Check-Act)    Based Thinking)
       │             │             │
       └────────┬────┴───────┬─────┘
                │            │
                ▼            ▼
           4.4 过程      6.1 风险和机遇
           8.1 运行       10.2 纠正措施
                          + 关联回 6.1
```

> **面试金句**：ISO 9001 的三大支柱是过程方法、PDCA、风险思维。三者不是并列的——**PDCA 是过程方法的"发动机"**，**风险思维是过程方法的"导航雷达"**。

---

## 二、PDCA 戴明环（**必背**）

PDCA = Plan–Do–Check–Act，由质量管理大师 **戴明（Deming）** 推广，又叫"戴明环"。

```
        ┌──────────┐
        │   Plan   │  策划：定目标、定方法
        └────┬─────┘
             ↓
   ┌──────────┐    ┌──────────┐
   │   Act    │ ←→ │    Do    │
   │  改进    │    │  执行    │
   └──────────┘    └────┬─────┘
             ↑          ↓
             ┌──────────┐
             │  Check   │
             │  检查    │
             └──────────┘
```

> 戴明名言：**"In God we trust, all others bring data."（质量要靠数据，不是靠感觉）**

---

## 三、PDCA 落到 ISO 9001 章节的对应

| PDCA | 含义 | 标准对应条款 | 文控岗动作 |
|---|---|---|---|
| **P**lan | 策划 | **6 章（6.1 风险 + 6.2 目标 + 6.3 变更）** + 7.5.2 创建 | 风险登记表、质量目标分解、模板提供 |
| **D**o | 执行 | **7 章（7.1-7.4 资源能力）+ 8 章（运行）** + 7.5.3 控制分发 | **文件创建、发放、培训记录** |
| **C**heck | 检查 | **9 章（9.1 监视 + 9.2 内审 + 9.3 管理评审）** | **内审、管理评审、数据分析** |
| **A**ct | 改进 | **10 章（10.1 改进 + 10.2 纠正措施 + 10.3 持续改进）** | 不合格报告、CAPA 闭环归档 |

**记忆法**：**P**lan → **6**；**D**o → **7/8**；**C**heck → **9**；**A**ct → **10**

## 四、PDCA 的 4 步 + 8 阶段（戴明原版·细化）

| 阶段 | PDCA | 解释 |
|---|---|---|
| 1 | P | 找出问题 |
| 2 | P | 分析原因 |
| 3 | P | 找出主要原因 |
| 4 | P | 制定对策 |
| 5 | D | 执行对策 |
| 6 | C | 检查效果 |
| 7 | A | 巩固措施 |
| 8 | A | 遗留问题 → 进入下一个 PDCA |

**面试金句**：PDCA 不是一次性的，它是一个**螺旋上升**的环，每转一圈问题就被解决一部分，然后发现下一层问题，再转一圈。

---

## 五、PDCA × 过程方法落地模板（**实操**）

| PDCA 阶段 | 标准条款 | 落地动作 | 产出物 |
|---|---|---|---|
| P | **6.2** | 质量目标 SMART + 准则 + 资源 + 责任人 + 时间表 | 目标分解表 |
| D | **8.1** | 按准则执行 + 记录关键参数 | 过程运行记录 |
| C | **9.1** | KPI 监视测量 + 数据分析（**9.1.3 七件套**） | KPI 报表 |
| A | **10.1-10.3** | 偏差纠正 + 原因分析 + 措施 + 有效性评审 | CAPA 报告 |

> **风险点**：PDCA 流程化容易变成形式主义——**必须把 KPI 挂到部门/个人绩效考核**（这是 cheatsheet 提醒的"trade-off"）。

## 六、过程方法（Process Approach）

ISO 9001:2015 把"过程方法"作为七大原则之一（标准 0.2 + 第 4 原则）。

### 1. 什么是过程
**过程 = 输入 + 活动 + 输出**（任何工作都是过程）

**举例**："处理客户投诉"是一个过程
- **输入**：客户投诉单
- **活动**：受理 → 记录 → 调查原因 → 制定对策 → 回复客户
- **输出**：投诉处理报告 + CAPA 措施

### 2. 过程的相互作用（PDCA 在过程中的体现）

```
客户要求 ─→ [过程A] ─→ 输出1 ─→ [过程B] ─→ 输出2 ─→ 客户满意
              ↓           ↓          ↓          ↓
              监控 ←────  监控  ←─── 监控  ←─── 监控
              ↑
            改进（PDCA + 风险思维）
```

## 七、SIPOC（**比乌龟图更适合面试**）

ISO 9001 推荐用 **SIPOC** 描述一个过程，比"乌龟图"更标准化。

| 字母 | 英文 | 中文 | 含义 | 例子（文件管控过程） |
|---|---|---|---|---|
| **S** | Suppliers | 供方 | 谁提供输入？ | 各业务部门、外审机构 |
| **I** | Inputs | 输入 | 需要什么？ | 客户标准、法规、内部需求 |
| **P** | Process | 过程 | 怎么做？ | 编号 → 审核 → 批准 → 发放 → 回收 |
| **O** | Outputs | 输出 | 产出什么？ | 受控文件、签收记录 |
| **C** | Customers | 顾客 | 给谁用？ | 现场操作人员、外审员 |

> **面试用 SIPOC 描述"文控工作"比乌龟图更专业**——审核员和体系工程师都在用 SIPOC。

## 八、COP 乌龟图（**次选·国内更常见**）

```
                 ┌─────────────────────────┐
                 │   过程名称（乌龟壳）     │
                 │  e.g. 文件管控过程       │
                 └─────────────────────────┘
                
    输入  ←  资源 / 准则  →  输出
    ↑                              ↑
    │   ┌───────────┐              │
    │   │  关键活动  │              │
    │   │  (乌龟腿)  │              │
    │   └───────────┘              │
    ↑                              ↑
    责任部门       KPI 考核      客户（内/外）
    (乌龟头)    (乌龟尾巴)      (乌龟尾)
```

### 乌龟图 6 问（与 SIPOC 互补）

| 要素 | 问题 | 例子（文件管控过程） |
|---|---|---|
| **输入** | 过程需要什么？ | 客户标准、法规、内部需求 |
| **输出** | 过程产出什么？ | 受控文件、签收记录 |
| **准则** | 用什么标准做？ | ISO 9001 7.5、公司文件管理办法 |
| **资源** | 需要什么人/设备？ | 文控员 + ERP + NAS |
| **责任（Owner）** | 谁负责？ | 文控员 → 体系经理 |
| **KPI** | 怎么衡量？ | 文件及时归档率 ≥ 99%、过期文件回收率 100% |
| **风险/机遇** | 哪些会出错？ | 版本混用、外发失控（**对应 6.1**） |

## 九、4.4 过程清单与过程图（**QMS 必做**）

**当用到**：QMS 建立、年度评审、内审准备。

**做法**：建一张主表 + 一张过程关系图。

### 主表示例

| 过程名 | 输入 | 输出 | 准则 | KPI | 责任 | 主要风险 | 条款 |
|---|---|---|---|---|---|---|---|
| 文件管控 | 业务需求、法规 | 受控文件 | 7.5、文件管理办法 | 归档及时率 | 文控员 | 版本混用 | 7.5.3 |
| 客户投诉 | 投诉单 | 处理报告 | 8.2、10.2 | 关闭率 | 客服 | 升级延迟 | 10.2 |
| 供方管理 | 采购需求 | 合格供方 | 8.4 | 供方绩效 | 采购 | 供方质量 | 8.4 |
| ... | ... | ... | ... | ... | ... | ... | ... |

### 关系图（核心+支持+管理）

```
核心过程:  顾客需求 → 设计 → 生产 → 交付
支持过程:  培训、设备、采购、文件
管理过程:  管理评审、内审、CAPA
```

> **过程数量建议**：一般 **10–20 个**为宜；行业细分可加子过程。30+ 过程会难以维护。

## 十、用 PDCA 解释"文件管控"全过程（**面试杀手锏**）

> 面试 Q：你的文控工作怎么结合 PDCA？

**答**：
- **P（策划）**：建立文件管理办法（编号规则、版本控制规则）、识别 6.1 风险和机遇（如版本混用、外发失控）、制定质量目标（如"归档及时率 ≥ 99%"）
- **D（执行）**：按 SOP 创建、审核、批准文件，发放到现场，做培训签到；按 7.5.3 控制分发和访问
- **C（检查）**：内审（9.2）时检查现场文件是否最新、是否有效标识、不符合项整改率；管理评审（9.3）输入相关 KPI
- **A（改进）**：根据审核结果修订文件管理办法，更新模板，启动 CAPA（10.2 六步循环），**回写 6.1 风险表**（新发现的风险）

## 十一、基于风险的思维（Risk-Based Thinking）—— **2015 版新支柱**

### 1. 是什么
- 在策划和实施 QMS 时**主动考虑风险和机遇**，而不是事后补救
- 对应 **6.1** 条款 + 10.2.1 e（不合格后回写 6.1）

### 2. 2008 → 2015 的关键升级

| 2008 版 | 2015 版 |
|---|---|
| 8.5.3 预防措施（独立条款） | **6.1 风险和机遇**（贯穿全标准） |
| 被动应对 | 主动识别 |

### 3. 6.1 应对风险的方式（5 种·必背）

| 方式 | 含义 | 例子 |
|---|---|---|
| **规避** | 远离风险源 | 不做该业务 |
| **改变可能性** | 降低发生概率 | 双人复核、防呆设计 |
| **改变后果** | 降低影响程度 | 应急响应、备份 |
| **分担** | 转嫁风险 | 买保险、外包 |
| **通过信息充分决策保留** | 知情接受 | 高层决策接受某种风险 |

### 4. 机遇的方式（6 种）

采用新实践、推出新产品、开辟新市场、赢得新顾客、建立伙伴关系、利用新技术

> ⚠️ **标准译法**：6.1 是"**应对风险和机遇的措施**"——"机遇"不是"机会"。

## 十二、面试高频问题

### Q1：PDCA 与 ISO 9001 怎么对应？

**答**：P→第 6 章策划；D→第 7+8 章支持与运行；C→第 9 章绩效评价；A→第 10 章改进。每个子过程都嵌入 PDCA。

### Q2：过程方法、PDCA、风险思维是什么关系？

**答**：ISO 9001:2015 引言 0.3 明确这是三大支柱。过程方法是骨架，PDCA 是发动机的循环，风险思维是导航雷达。**PDCA 是过程方法的具体化**，**风险思维把 2008 版的"预防措施"升级为内嵌能力**。

### Q3：怎么描述一个过程？

**答**：用 SIPOC（供方/输入/过程/输出/顾客），或者乌龟图（6 问）。SOP 编制时一定要画 SIPOC，确保过程边界清晰。

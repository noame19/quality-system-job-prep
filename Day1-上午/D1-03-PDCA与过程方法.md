## 一、PDCA 是什么（戴明环）

PDCA = **Plan–Do–Check–Act**，由质量管理大师 **戴明（Deming）** 推广，又叫"戴明环"。

```
        ┌──────────┐
        │   Plan   │  策划：定目标、定方法
        └────┬─────┘
             ↓
   ┌──────────┐    ┌──────────┐
   │   Act    │ ←→ │    Do    │
   │  改进    │    │  执行    │
   └──────────┘    └────┬─────┘
             ↑          ↓
             ┌──────────┐
             │  Check   │
             │  检查    │
             └──────────┘
```

> 戴明说过一句名言：**"In God we trust, all others bring data."（质量要靠数据，不是靠感觉）**

---

## 二、PDCA 落到 ISO 9001 的对应章节

| PDCA | 含义 | ISO 9001 对应 | 文控岗相关 |
|---|---|---|---|
| **P**lan | 策划 | 第 4、5、6 章 | 风险评估表、质量目标分解 |
| **D**o | 执行 | 第 7、8 章 | **文件创建、发放、培训记录** |
| **C**heck | 检查 | 第 9 章 | **内部审核、管理评审、监视测量** |
| **A**ct | 改进 | 第 10 章 | 不合格品报告、CAPA |

**记忆法**：**P**lan 策 → **D**o 行 → **C**heck 查 → **A**ct 改

---

## 三、PDCA 的 4 步 + 8 阶段（戴明原版）

| 阶段 | PDCA | 解释 |
|---|---|---|
| 1 | P | 找出问题 |
| 2 | P | 分析原因 |
| 3 | P | 找出主要原因 |
| 4 | P | 制定对策 |
| 5 | D | 执行对策 |
| 6 | C | 检查效果 |
| 7 | A | 巩固措施 |
| 8 | A | 遗留问题 → 进入下一个 PDCA |

**面试金句**：PDCA 不是一次性的，它是一个**螺旋上升**的环，每转一圈问题就被解决一部分，然后发现下一层问题，再转一圈。

## 四、过程方法（Process Approach）

ISO 9001:2015 把"过程方法"作为七大原则之一。

### 1. 什么是过程
**过程 = 输入 + 活动 + 输出**（任何工作都是过程）

**举例**："处理客户投诉"是一个过程
- 输入：客户投诉单
- 活动：受理 → 记录 → 调查原因 → 制定对策 → 回复客户
- 输出：投诉处理报告 + CAPA 措施

### 2. 过程的相互作用（PDCA 在过程中的体现）

```
客户要求 ─→ [过程A] ─→ 输出1 ─→ [过程B] ─→ 输出2 ─→ 客户满意
              ↓           ↓          ↓          ↓
              监控 ←────  监控  ←─── 监控  ←─── 监控
              ↑
            改进（PDCA）
```

## 五、COP 乌龟图（Process Turtle）—— 描述一个过程的工具

ISO 9001 标准推荐的"一张图说清一个过程"的方法，画法像乌龟：

```
                 ┌─────────────────────────┐
                 │   过程名称（乌龟壳）     │
                 │  e.g. 文件管控过程       │
                 └─────────────────────────┘
                
    输入  ←  资源 / 准则  →  输出
    ↑                              ↑
    │   ┌───────────┐              │
    │   │  关键活动  │              │
    │   │  (乌龟腿)  │              │
    │   └───────────┘              │
    ↑                              ↑
    责任部门       KPI 考核      客户（内/外）
    (乌龟头)    (乌龟尾巴)      (乌龟尾)
```

### 6 大要素（6 问）

| 要素 | 问题 | 例子（文件管控过程） |
|---|---|---|
| **输入（Input）** | 过程需要什么？ | 客户标准、法规、内部需求 |
| **输出（Output）** | 过程产出什么？ | 受控文件、签收记录 |
| **准则（Criteria）** | 用什么标准做？ | ISO 9001 7.5、公司文件管理办法 |
| **资源（Resources）** | 需要什么人/设备？ | 文控员 + ERP + NAS |
| **责任（Owner）** | 谁负责？ | 文控员 → 体系经理 |
| **KPI** | 怎么衡量？ | 文件及时归档率 ≥ 99%、过期文件回收率 100% |
| **风险/机遇（Risk）** | 哪些会出错？ | 版本混用、外发失控 |

## 六、用 PDCA 解释"文件管控"全过程（面试杀手锏）

> 面试 Q：你的文控工作怎么结合 PDCA？

**答**：
- **P（策划）**：建立文件管理办法、编号规则、版本控制规则、培训计划
- **D（执行）**：按 SOP 创建、审核、批准文件，发放到现场，做培训签到
- **C（检查）**：内审时检查现场文件是否最新、是否有效标识、不符合项整改率
- **A（改进）**：根据审核结果修订文件管理办法，更新模板，进入下一轮 PDCA

## 八、自测题

1. PDCA 4 个字母分别代表什么？每步 1 句话解释。
2. PDCA 与 ISO 9001 第 4–10 章的对应关系是什么？
3. 描述一个"品质周报编制"过程（用 6 要素乌龟图）。
4. 为什么说 PDCA 是螺旋上升而不是直线？

**参考答案**：
1. P 策划（定目标/方法）、D 执行（按计划做）、C 检查（对照目标看效果）、A 改进（巩固/修正）
2. P→4/5/6、D→7/8、C→9、A→10
3. 略（练习）
4. 每转一圈解决问题+发现新问题，所以是螺旋上升

## 一、文件四层架构（**最重要**）

ISO 9001 要求的"文件化信息"在企业中通常按 4 个层级组织：

```
┌─────────────────────────────────────────────┐
│ Level 1  质量手册（Quality Manual）          │  ← 顶层纲领
│          1 份 / 公司，描述整个体系范围        │
├─────────────────────────────────────────────┤
│ Level 2  程序文件（Procedures）               │  ← 二级规范
│          描述"谁来做什么、为什么做、记录在哪"  │
├─────────────────────────────────────────────┤
│ Level 3  作业指导书 SOP/WI（Work Instruction）│  ← 三级操作
│          现场人员"手把手怎么干"              │
├─────────────────────────────────────────────┤
│ Level 4  表单/记录（Forms/Records）           │  ← 四级证据
│          检验报告、培训签到、不合格报告         │
└─────────────────────────────────────────────┘
```

### 各层级的特点和数量

| 层级 | 文件类型 | 数量级 | 审批人 | 文控岗动作 |
|---|---|---|---|---|
| Level 1 | 质量手册 | 1 份 | 总经理 | 受控、分发到管理层 |
| Level 2 | 程序文件 | 10–30 份 | 部门经理 | 编号、受控、归档 |
| Level 3 | SOP/WI | 50–300 份 | 主管/工程师 | 编号、张贴在现场 |
| Level 4 | 表单/记录 | 上百种 | 文员/主管 | 收集、保存、销毁 |

### 通俗比喻（讲给 HR 听）

> 质量手册像宪法 → 程序文件像法律 → SOP 像地方法规 → 表单像判决书

## 二、SOP vs WI vs 流程图 的关系（**高频面试题**）

| 类型 | 全称 | 范围 | 详细度 |
|---|---|---|---|
| **SOP** | Standard Operating Procedure 标准作业程序 | 整个过程（如"来料检验 SOP"） | 中等，1–5 页 |
| **WI** | Work Instruction 作业指导书 | 单一步骤（如"拉力测试 WI"） | 详细，含图示 |
| **流程图** | Flowchart | 过程流向 | 简化版，1–2 页 |

**关系**：SOP 包含多个 WI，SOP 告诉你"做什么"，WI 告诉你"怎么做"。

> 面试金句：**SOP 是骨架，WI 是血肉，流程图是导航。**

## 三、文件编号规则（**必掌握**）

### 1. 典型编号结构

```
A - BBB - CCCC - D . E
│   │     │     │   └─ 版本号 Vx.x
│   │     │     └───── 文件顺序号
│   │     └─────────── 文件类别代码
│   └───────────────── 部门代码
└───────────────────── 公司代码（可省略）
```

### 2. 部门代码（参考）

| 代码 | 部门 | 代码 | 部门 |
|---|---|---|---|
| ADM | 行政部 | QA | 品保部 |
| HR | 人事部 | QC | 品检部 |
| FIN | 财务部 | PRD | 生产部 |
| PUR | 采购部 | ENG | 工程部 |
| SAL | 销售部 | WH | 仓库 |
| LOG | 物流 | DCC | 文控中心 |

### 3. 文件类别代码（参考）

| 代码 | 类别 | 例子 |
|---|---|---|
| QM | 质量手册 | QM-001-V1.0 |
| PR | 程序文件 | QA-PR-001-V1.0（检验管理程序） |
| SOP | 标准作业程序 | QC-SOP-005-V2.0（来料检验 SOP） |
| WI | 作业指导书 | QC-WI-012-V1.0（卡尺测量 WI） |
| FRM | 表单 | QA-FRM-008-V1.0（首件检验记录表） |
| REC | 记录 | QA-REC-2024-001（年度内审报告） |
| ECN | 工程变更 | ECN-2024-015 |
| SPEC | 规格书 | ENG-SPEC-AC-Power-Cord-V3.0 |

### 4. 版本号规则

| 变更类型 | 版本号变化 | 例子 |
|---|---|---|
| 文字/格式微调 | 小版本 +0.1 | V1.0 → V1.1 |
| 章节/工艺变更 | 大版本 +1.0 | V1.9 → V2.0 |
| 全新文件 | V1.0 | — |

**口诀**：**大改大跳、小改小跳**（2.0 → 3.0 是大改，2.0 → 2.1 是小改）

### 5. 受控标识

每份受控文件必须有"受控章"（蓝色公章），盖在文件首页右上角。  
常见样式：

```
╔════════════════╗
║   [受控副本]   ║
║  编号：C-001   ║
║  发放：生产部  ║
║  日期：2024-01 ║
╚════════════════╝
```

## 四、文件状态分类

| 状态 | 标识 | 说明 |
|---|---|---|
| **受控** | 蓝色受控章 | 经审批发放，正在生效 |
| **非受控** | 红色"仅供参考" | 用于客户/外部参考，不随公司变更 |
| **作废** | 红色"作废"章 | 已废止，需回收或销毁 |
| **保密** | 加密/水印 | 受密级管控（机密/秘密/绝密） |

## 五、外来文件管控（容易被忽视）

**外来文件 = 非本公司编制的文件**，但要纳入体系管控。例子：

- 客户标准（如沃尔玛验厂标准）
- 国家标准（如 GB 4706.1 家电安全）
- 行业标准（如 UL 62 电线电缆）
- 法规（如《产品质量法》）
- 上级集团下发的管理手册

**管控要求**：
1. 收集（关注法规更新、行业新版本）
2. 编号（标为 EXT-xxx）
3. 发放到相关部门
4. 定期评审有效性
5. 作废时回收

## 六、自创"XX 电源线公司文件编号规则"（**面试作品**）

> **假设公司：东莞 X 电源线有限公司**，主营 AC 电源线

```yaml
编号格式: AA-BBB-CCCC-D.E
AA    : 公司代码 (X = X 公司)
BBB   : 部门代码 (QA/QC/PRD/ENG/HR/...)
CCCC  : 文件类别代码
D     : 顺序号 (3 位数字 001-999)
E     : 版本号 (Vx.x)

示例:
  X-QA-PR-001-V1.0   # 检验管理程序
  X-QC-SOP-005-V2.0  # 来料检验 SOP（第 2 版）
  X-ENG-SPEC-001-V3.0 # AC 电源线产品规格书（第 3 版）
  X-QA-FRM-008-V1.0  # 首件检验记录表
  X-ECN-2024-015      # 2024 年第 15 号工程变更

受控章样式: 
  蓝色公章，标"受控副本+编号+发放范围+生效日期"

版本号:
  V1.0 → V1.1 → V1.2 ... V2.0（V2.0 是大改）

文件状态:
  受控 / 非受控 / 作废 / 保密
```

## 九、自测题（必做）

1. 文件四层架构是哪 4 层？每层 1 个例子。
2. 编号 "QA-SOP-005-V2.0" 中 QA、SOP、005、V2.0 分别代表什么？
3. SOP 和 WI 的本质区别是什么？
4. 大版本号（V1.0 → V2.0）和小版本号（V1.0 → V1.1）分别在什么情况下用？

**参考答案**：
1. L1 质量手册、L2 程序文件、L3 SOP/WI、L4 表单/记录
2. QA = 部门、SOP = 类别、005 = 顺序号、V2.0 = 第 2 版
3. SOP 是"过程"，WI 是"步骤"；SOP 包含多个 WI
4. 大改=结构性变更（如工艺路线变了），小改=文字/格式微调

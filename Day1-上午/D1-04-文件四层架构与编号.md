## 一、为什么 7.5 是文控岗的"唯一标准"？

> ⚠️ 旧课件常把"文件管控"笼统说成"第 7 章"——不准确。
> 第 7 章是**支持**（5M1E + 成文信息），文控岗只对应 **7.5 成文信息** 这一节。

ISO 9001:2015 的 7.5 节包含 **3 个子条款**：

| 子条款 | 标题 | 核心问题 |
|---|---|---|
| **7.5.1** | 总则 | 哪些成文信息必须保留？ |
| **7.5.2** | 创建和更新 | 创建新文件时要做什么？ |
| **7.5.3** | 成文信息的控制 | 怎么分发、保护、变更、保留、处置？ |

---

## 二、7.5.1 总则（**必背**）

> 组织应包括**两方面**成文信息：

### a) 本标准要求的成文信息
- 4.3 QMS 范围
- 5.2 质量方针
- 6.1.1 风险和机遇（**风险登记表**）
- 6.2 质量目标
- 7.2 d）能力证据（培训记录）
- 8.1 运行策划
- 8.3.3 设计和开发输入
- 8.4.1 外部供方控制类型和程度
- 8.5.1 生产和服务提供的控制
- 8.5.2 标识和可追溯性
- 8.5.3 顾客或外部供方的财产
- 8.5.6 更改控制
- 8.6 放行证据
- 8.7 不合格性质和措施
- 9.1.1 监视测量结果分析证据
- 9.2.2 审核方案和结果
- 9.3 管理评审结果证据
- 10.2.2 不合格性质和措施证据

### b) 组织确定的 QMS 有效性所必需的成文信息
- SOP、WI、流程图、模板
- 内审检查表、CAPA 表
- 工艺参数表、BOM

> ⚠️ **审核员追问**："为什么这份 SOP 存在？"——你必须能用 7.5.1 a) 或 b) 解释清楚。

## 三、7.5.2 创建和更新（**必背**）

> 创建和更新成文信息时，组织应确保适当的：

| 要求 | 含义 | 文控岗动作 |
|---|---|---|
| **a) 标识和说明** | 标题、日期、作者、参考编号 | 提供文件编号规则；每份文件首页有这 4 项 |
| **b) 形式** | 语言、软件版本、图表 | 统一用中文 / Word / Visio 等 |
| **c) 评审和批准** | 适宜性和充分性 | 跨部门评审 + 授权人批准 |

### 文件首页标准格式

```
┌─────────────────────────────────────┐
│  XX 公司 SOP 文件                    │
├─────────────────────────────────────┤
│  标题：来料检验 SOP                  │
│  文件编号：QC-SOP-005               │
│  版本：V1.0                         │
│  日期：2024-06-20                   │
│  作者：张三（工程师）               │
│  审核：李四（QA 主管）              │
│  批准：王五（质量经理）             │
│  生效日期：2024-07-01               │
└─────────────────────────────────────┘
```

> **面试金句**：7.5.2 强调三个动作——**标识、形式、评审批准**。我作为文控岗会确保公司所有文件首页都包含标题、日期、作者、参考编号这 4 个标识项。

## 四、7.5.3 成文信息的控制（**核心战场·必背**）

> 7.5.3.1 控制范围：a) 可获得（适用时）；b) 充分保护

### a) 可获得（适用时）
> 意味着："对需要的人能找到"。文控岗要确保：
- 现场使用的 SOP 是最新版
- 电子件有访问权限管理
- 纸质件有受控标识

### b) 充分保护（**6 个具体动作**）

| # | 动作 | 含义 | 文控岗动作 |
|---|---|---|---|
| ① | **保密** | 不外泄 | 密级标识、外发审批 |
| ② | **防止丢失** | 不丢失 | 备份（3-2-1）+ 受控章 |
| ③ | **保持完整性** | 不被篡改 | 电子文件权限 + 纸质盖章 |
| ④ | **分发** | 怎么发 | 发放登记表、签字确认 |
| ⑤ | **访问** | 谁可以看 | 权限分级 |
| ⑥ | **检索** | 怎么找 | 文件总清单 + 命名规则 |
| ⑦ | **可用性** | 还能读 | 长期保存格式（如 PDF/A） |
| ⑧ | **迁移** | 平台升级 | 定期格式转换 |

### 7.5.3.2 控制分发访问（防止无权访问）

- **电子文件**：ERP/PLM 系统按角色分配权限
- **纸质文件**：受控章 + 发放登记
- **外来文件**：标"非受控"用于外部参考

### 7.5.3.3 存储和保护的变更控制

- **存储**：防失真、防潮、防虫（纸质）；防病毒、防误删（电子）
- **变更**：版本控制（V1.0 → V1.1 → V2.0），ECN 流程

### 7.5.3.4 保留期 + 处置

| 文件类型 | 保留期 |
|---|---|
| 质量手册 / 程序文件 | 永久（直到被新版替代 + 5 年） |
| SOP / WI | 现行 + 旧版保留 ≥ 3 年 |
| 品质记录（IQC/IPQC/OQC） | ≥ 3 年（部分 ≥ 5 年） |
| RoHS 测试报告 | ≥ 5 年（部分 ≥ 10 年） |
| 内审 / 外审报告 | ≥ 5 年 |
| 管理评审记录 | ≥ 5 年 |
| 培训记录 | 在职 + 离职后 3 年 |
| 客户投诉记录 | ≥ 5 年 |
| 设备校准证书 | 设备使用期 + 5 年 |
| ECN 工程变更 | 永久 |

> **处置**：期满后 → 部门主管审核 → 体系经理审批 → 监销 → 销毁记录保留 ≥ 5 年。

## 五、文件四层架构（**国内主流·与 7.5 不冲突**）

> **重要前提**：ISO 9001:2015 **不强制文件结构**（附录 A.4 明确）——你仍可用"手册/程序/记录"三级，只要满足 7.5 的内容要求。

```
┌─────────────────────────────────────────────┐
│ Level 1  质量手册（Quality Manual）          │  ← 顶层纲领（1 份 / 公司）
│          描述整个体系范围（4.3）             │
├─────────────────────────────────────────────┤
│ Level 2  程序文件（Procedures）               │  ← 二级规范（10–30 份）
│          描述"谁来做什么、为什么做、记录在哪"  │
├─────────────────────────────────────────────┤
│ Level 3  作业指导书 SOP/WI                    │  ← 三级操作（50–300 份）
│          现场人员"手把手怎么干"               │
├─────────────────────────────────────────────┤
│ Level 4  表单/记录（Forms/Records）           │  ← 四级证据（上百种）
│          检验报告、培训签到、不合格报告         │
└─────────────────────────────────────────────┘
```

### 各层级的特点和数量

| 层级 | 文件类型 | 数量级 | 审批人 | 文控岗动作 |
|---|---|---|---|---|
| Level 1 | 质量手册 | 1 份 | 总经理 | 受控、分发到管理层 |
| Level 2 | 程序文件 | 10–30 份 | 部门经理 | 编号、受控、归档 |
| Level 3 | SOP/WI | 50–300 份 | 主管/工程师 | 编号、张贴在现场 |
| Level 4 | 表单/记录 | 上百种 | 文员/主管 | 收集、保存、销毁 |

### 通俗比喻（讲给 HR 听）

> 质量手册像宪法 → 程序文件像法律 → SOP 像地方法规 → 表单像判决书

> ⚠️ **避免误区**：
> - 标准结构（10 章）≠ 组织文件结构（4 层）
> - 内部术语可保留——但要在文件中说明与标准术语的对应（cheatsheet 第 12 项误区）

## 六、SOP vs WI vs 流程图 的关系（**高频面试题**）

| 类型 | 全称 | 范围 | 详细度 |
|---|---|---|---|
| **SOP** | Standard Operating Procedure 标准作业程序 | 整个过程（如"来料检验 SOP"） | 中等，1–5 页 |
| **WI** | Work Instruction 作业指导书 | 单一步骤（如"拉力测试 WI"） | 详细，含图示 |
| **流程图** | Flowchart | 过程流向 | 简化版，1–2 页 |

**关系**：SOP 包含多个 WI，SOP 告诉你"做什么"，WI 告诉你"怎么做"。

> **面试金句**：**SOP 是骨架，WI 是血肉，流程图是导航。**

## 七、文件编号规则（**必掌握**）

### 1. 典型编号结构

```
A - BBB - CCCC - D . E
│   │     │     │   └─ 版本号 Vx.x
│   │     │     └───── 文件顺序号
│   │     └─────────── 文件类别代码
│   └───────────────── 部门代码
└───────────────────── 公司代码（可省略）
```

### 2. 部门代码（参考）

| 代码 | 部门 | 代码 | 部门 |
|---|---|---|---|
| ADM | 行政部 | QA | 品保部 |
| HR | 人事部 | QC | 品检部 |
| FIN | 财务部 | PRD | 生产部 |
| PUR | 采购部 | ENG | 工程部 |
| SAL | 销售部 | WH | 仓库 |
| LOG | 物流 | DCC | 文控中心 |

### 3. 文件类别代码（参考）

| 代码 | 类别 | 例子 |
|---|---|---|
| QM | 质量手册 | QM-001-V1.0 |
| PR | 程序文件 | QA-PR-001-V1.0（检验管理程序） |
| SOP | 标准作业程序 | QC-SOP-005-V2.0（来料检验 SOP） |
| WI | 作业指导书 | QC-WI-012-V1.0（卡尺测量 WI） |
| FRM | 表单 | QA-FRM-008-V1.0（首件检验记录表） |
| REC | 记录 | QA-REC-2024-001（年度内审报告） |
| ECN | 工程变更 | ECN-2024-015 |
| SPEC | 规格书 | ENG-SPEC-AC-Power-Cord-V3.0 |

### 4. 版本号规则

| 变更类型 | 版本号变化 | 例子 |
|---|---|---|
| 文字/格式微调 | 小版本 +0.1 | V1.0 → V1.1 |
| 章节/工艺变更 | 大版本 +1.0 | V1.9 → V2.0 |
| 全新文件 | V1.0 | — |

**口诀**：**大改大跳、小改小跳**（2.0 → 3.0 是大改，2.0 → 2.1 是小改）

### 5. 受控标识

每份受控文件必须有"受控章"（蓝色公章），盖在文件首页右上角。  
常见样式：

```
╔════════════════╗
║   [受控副本]   ║
║  编号：C-001   ║
║  发放：生产部  ║
║  日期：2024-01 ║
╚════════════════╝
```

## 八、文件状态分类

| 状态 | 标识 | 7.5.3 对应动作 |
|---|---|---|
| **受控** | 蓝色受控章 | 受控章 + 发放登记 |
| **非受控** | 红色"仅供参考" | 用于客户/外部参考 |
| **作废** | 红色"作废"章 | 隔离 + 保留期满销毁 |
| **保密** | 加密/水印 | 密级标识 + 访问控制 |

## 九、外来文件管控（**容易被忽视**）

**外来文件 = 非本公司编制的文件**，但要纳入 7.5.3 控制。例子：

- 客户标准（如 Walmart 验厂标准）
- 国家标准（如 GB 4706.1 家电安全）
- 行业标准（如 UL 62 电线电缆）
- 法规（如《产品质量法》）
- 上级集团下发的管理手册

**管控要求**：
1. 收集（关注法规更新、行业新版本）
2. 编号（标为 EXT-xxx）
3. 发放到相关部门
4. 定期评审有效性
5. 作废时回收

## 十、自创"XX 电源线公司文件编号规则"（**面试作品**）

> **假设公司：东莞 X 电源线有限公司**，主营 AC 电源线

```yaml
编号格式: AA-BBB-CCCC-D.E
AA    : 公司代码 (X = X 公司)
BBB   : 部门代码 (QA/QC/PRD/ENG/HR/...)
CCCC  : 文件类别代码
D     : 顺序号 (3 位数字 001-999)
E     : 版本号 (Vx.x)

示例:
  X-QA-PR-001-V1.0   # 检验管理程序

  蓝色公章，标"受控副本+编号+发放范围+生效日期"

版本号:
  V1.0 → V1.1 → V1.2 ... V2.0（V2.0 是大改）

文件状态:
  受控 / 非受控 / 作废 / 保密

7.5.2 文件首页必含:
  标题、日期、作者、参考编号
```

## 十二、面试常见误区清单（**避免被反问打脸**）

| 误区 | 正确做法 |
|---|---|
| 把"手册/程序/记录"三级当标准要求 | 标准只要求 7.5 内容，不强制结构 |
| 现场只有受控章，无受控副本号 | 受控章 + 编号 + 发放范围 + 生效日期 |
| 电子文件没版本控制 | 用 ERP/SharePoint/VSS 做版本管理 |
| 外来文件未识别（7.5.3 d） | 标"非受控"或纳入受控清单 |
| 旧版文件没回收 | 严格走 7.5.3 变更控制 |
| 保留期满未处置 | 部门主管审核 → 体系经理审批 → 监销 |

## 十三、自测题（必做）

1. 7.5 有几个子条款？分别讲什么？
2. 7.5.1 包含哪两类成文信息？
3. 7.5.2 创建和更新时要满足的 3 个要求是什么？
4. 7.5.3 控制的 6 个具体动作（b 充分保护）是什么？
5. 标准 10 章结构 = 公司文件结构吗？为什么？

**参考答案**：

**1.** 3 个：7.5.1 总则；7.5.2 创建和更新；7.5.3 成文信息的控制

**2.** a) 标准要求的成文信息；b) 组织确定的 QMS 有效性所必需的成文信息

**3.** a) 标识和说明（标题、日期、作者、参考编号）；b) 形式（语言、软件版本、图表）；c) 评审和批准

**4.** 保密、防止丢失、保持完整性、分发、访问、检索（**含可用性、迁移**）

**5.** 不等。标准 10 章是"做什么"的框架，公司文件结构可以自己设计（手册/程序/记录三级只是国内常用做法之一），但必须满足 7.5 内容要求。

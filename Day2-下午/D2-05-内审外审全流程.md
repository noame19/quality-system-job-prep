# D2-05｜内审 / 外审全流程（**精准 9.2 / 9.3 条款版**）

> ⏱ 时长：90 min（14:00–15:30）
> 🎯 目标：能讲清 **9.2 内审方案** 9 要素 + **9.3.2 管理评审 9 项输入** + **9.3.3 管理评审 3 件输出**

## 一、什么是审核（GB/T 19000 定义）

| 类型 | 主体 | 目的 | 频率 |
|---|---|---|---|
| **第一方审核（内审）** | 公司自己审自己 | 检查体系是否有效运行 | 每年 1 次全覆盖 |
| **第二方审核（客户验厂）** | 客户来审 | 评估是否能成为供应商 | 按客户要求 |
| **第三方审核（外审）** | 认证机构（SGS/BSI/TÜV） | 颁发 / 监督认证证书 | 每年 1 次监督 + 3 年 1 次复评 |

## 二、9.2 内部审核（**必背**）

### 9.2.1 目的（标准原文）

按策划的时间间隔进行内部审核，应提供有关 QMS 的下列信息：

- **a)** 是否符合组织自身的 QMS 要求
- **b)** 是否符合本标准（GB/T 19001-2016）的要求
- **c)** 是否得到有效实施和保持

> 这 3 个目的是**外审必问的**——回答时要分别举例。

### 9.2.2 审核方案 9 要素（标准原文·必背）

> 组织应策划、建立、实施和保持审核方案，**包括**：

| # | 要素 | 含义 | 文控岗动作 |
|---|---|---|---|
| 1 | **频次** | 多久审一次 | 制定年度频次表（高风险半年，其他年度） |
| 2 | **方法** | 怎么审 | 现场审 / 远程审 / 文件审 |
| 3 | **职责** | 谁负责 | 体系部牵头 |
| 4 | **策划要求和报告** | 策划 + 报告格式 | 编制审核计划 + 报告模板 |
| 5 | **审核准则和范围** | 对照什么审 | 标准条款 + 公司文件 |
| 6 | **选择审核员 + 客观公正** | 谁审 | **审核员不能审自己部门** |
| 7 | **审核结果报告给相关管理者** | 报告给谁 | 部门负责人 + 最高管理者 |
| 8 | **及时采取适当的纠正和纠正措施** | 跟进 | 闭环到 10.2 |
| 9 | **保留成文信息** | 留证据 | 审核方案 + 报告归档 |

> **补充要求**：考虑 a) 过程重要性；b) 以往审核结果

### 内审 vs 管理评审 vs 监视测量的关系

```
9.1 监视测量（日常体检）
    ↓
9.2 内审（定期体检：半年/年度覆盖全部条款）
    ↓
9.3 管理评审（年度大体检：最高管理者主持，输入内审结果）
```

## 三、内审 7 步全流程（**实务**）

```
① 编制审核方案（年度）
       ↓
② 单次审核策划（具体部门）
       ↓
③ 编制审核检查表
       ↓
④ 现场审核（首次会议 + 现场 + 末次会议）
       ↓
⑤ 不符合项报告（NCR）
       ↓
⑥ 整改 + 验证（30 天）
       ↓
⑦ 审核报告 → 输入 9.3 管理评审
```

## 四、每步详解

### ① 编制年度审核方案（**核心**）
- **时间**：每年 12 月制定下一年度
- **内容**：覆盖 GB/T 19001-2016 所有条款（4–10 章）+ 所有部门
- **要素**：频次 + 方法 + 职责 + 策划要求 + 准则范围 + 审核员 + 报告对象 + 纠正措施 + 保留成文信息

**文控岗动作**：把《年度审核方案》存档，并发放到相关部门。

### ② 单次审核策划
- **审核组长**：有内审员证的人（建议 GB/T 19011 培训证书）
- **审核员**：每组 2-3 人，覆盖不同部门
- **文控岗**：作为"陪同员"或"记录员"参与

**内审员要求**：
- 接受过 ISO 9001 内审员培训（一般 16-24 学时）
- 通过考试（部分公司自己出题）
- 每年至少参加 1 次内审
- **不能审自己部门**

### ③ 编制审核检查表
- **格式**：按 ISO 9001 条款 + 部门
- **示例**（文控岗配合部分）：

```
┌─────────────────────────────────────────────────────────┐
│ GB/T 19001-2016 内审检查表（品保部 / 文控）              │
├──────┬──────────────┬──────────────┬──────────┬─────────┤
│ 条款 │ 审核要点     │ 查证方法     │ 审核发现 │ 判定    │
├──────┼──────────────┼──────────────┼──────────┼─────────┤
│7.5.1 │ 是否建立 7.5.1│ 查程序文件   │ 有 PR-001│ 符合    │
│      │ a/b 成文信息  │              │          │         │
├──────┼──────────────┼──────────────┼──────────┼─────────┤
│7.5.2 │ 文件创建更新 │ 抽查 5 份文件 │ V2.0 流程│ 符合    │
│      │ 是否规范     │ 看标识说明   │          │         │
├──────┼──────────────┼──────────────┼──────────┼─────────┤
│7.5.3 │ 受控标识     │ 看是否有蓝章 │ 都有     │ 符合    │
│      │ + 6 个保护动作│ 看权限+版本  │          │         │
├──────┼──────────────┼──────────────┼──────────┼─────────┤

├──────┼──────────────┼──────────────┼──────────┼─────────┤
│7.5.3 │ 旧版是否回收 │ 查回收登记   │ 已回收   │ 符合    │
├──────┼──────────────┼──────────────┼──────────┼─────────┤
│ ...  │ ...          │ ...          │ ...      │ ...     │
└──────┴──────────────┴──────────────┴──────────┴─────────┘
```

### ④ 现场审核（**重点**）

#### 4.1 首次会议（15 分钟）
- 审核组长主持
- 介绍审核组成员、审核目的（对应 9.2.1 三件事）、范围、日程
- 部门负责人出席

#### 4.2 现场审核（2-4 小时 / 部门）

**审核技巧**：
- **抽样**：随机抽 3-5 份记录 / 现场抽 3-5 处
- **问 5W2H**：What / Who / When / Where / Why / How / How much
- **顺向追踪**：从客户订单 → 评审 → 生产 → 出货 → 客后
- **逆向追踪**：从不合格品 → CAPA（10.2）→ 整改 → 关闭
- **5Why 分析**：一个问题连续问 5 个"为什么"找根本原因

#### 4.3 末次会议（15 分钟）
- 审核组长通报审核发现
- 宣读不符合项报告
- 与受审部门确认

### ⑤ 不符合项报告（NCR）

**3 类不符合**：

| 类型 | 严重程度 | 例子 |
|---|---|---|
| **严重不符合** | Major | 体系完全失效；多个条款违反 |
| **一般不符合** | Minor | 单点不符合 |
| **观察项 / 改进建议** | Observation | 潜在风险 |

**NCR 模板**：
```
不符合项报告 (NCR)
编号: NCR-2024-001
审核日期: 2024-06-25
受审部门: 仓库
条款: 7.5.3 成文信息的控制
描述: 仓库现场使用的 IQC SOP 是 V1.0 版本，但现行有效版本为 V1.1（2024-05-10 生效）
不符合标准: GB/T 19001-2016 7.5.3
不符合事实: 现场文件过期未回收
审核员: 张三          日期: 2024-06-25
部门负责人确认: 李四   日期: 2024-06-25
```

### ⑥ 整改 + 验证（**30 天内**）

**整改流程**（对应 10.2 六步循环，详见 D2-06）：
```
NCR 发出
    ↓
部门原因分析（5Why + 鱼骨图）
    ↓
制定纠正措施（短期 + 长期）
    ↓
实施整改
    ↓
提交证据（照片 / 新文件 / 培训记录）
    ↓
审核员验证有效性
    ↓
关闭 NCR 或退回重改
```

### ⑦ 审核报告
- **内容**：审核目的、范围、准则、发现、结论、建议
- **存档**：作为 **9.3 管理评审输入**（必查项）
- **文控岗动作**：归档到"内审"文件夹（保留 ≥ 5 年）

## 五、9.3 管理评审（**必背**）

### 9.3.1 总则（标准原文）
> 最高管理者应按策划的时间间隔对组织的 QMS 进行评审，**确保其持续的适宜性、充分性、有效性，并与组织的战略方向保持一致**。

**三件套**：
- **适宜性（Suitability）**：QMS 适合当前环境吗？
- **充分性（Sufficiency）**：资源够吗？
- **有效性（Effectiveness）**：达到预期结果了吗？

> ⚠️ **审核员追问**："管理评审的目的？"——答"适宜性、充分性、有效性 + 与战略一致"。

---

### 9.3.2 输入（**5 大类 9 项·必背**）

> ⚠️ 这是面试 + 外审的高频考点。

#### a) 以往管理评审所采取措施的情况
1. **上次管理评审决议跟进**（闭环情况）

#### b) 与 QMS 相关的内外部因素的变化（4.1）
2. **内外部因素变化**：原材料涨价、新法规、竞争对手新增等

#### c) 下列有关 QMS 绩效和有效性的信息（**七大件**）
3. **顾客满意和有关相关方的反馈**
4. **质量目标的实现程度**
5. **过程绩效以及产品和服务的合格情况**
6. **不合格及纠正措施**
7. **监视和测量结果**
8. **审核结果**
9. **外部供方的绩效**

> **额外第 8 项（必背）**：**应对风险和机遇所采取措施的有效性（包括其趋势）**——对应 6.1
> （这是 9.3.2 c) 的展开，原文是 9 项输入）

#### d) 资源的充分性
> （标准原文 d）项）

#### e) 改进的机会
> （标准原文 e）项）

### 9.3.2 输入完整列表（**10 项**）

| # | 输入项 | 来源条款 |
|---|---|---|
| 1 | 以往管理评审所采取措施的情况 | 9.3.2 a) |
| 2 | 内外部因素的变化 | 4.1 |
| 3 | 顾客满意和相关方反馈 | 9.1.2 |
| 4 | 质量目标的实现程度 | 6.2 |
| 5 | 过程绩效、产品和服务合格情况 | 9.1.1 |
| 6 | 不合格及纠正措施 | 10.2 |
| 7 | 监视和测量结果 | 9.1.1 |
| 8 | 审核结果 | 9.2 |
| 9 | 外部供方的绩效 | 8.4 |
| 10 | 应对风险和机遇所采取措施的有效性（含趋势） | 6.1 |
| 11 | 资源的充分性 | 7.1 |
| 12 | 改进的机会 | 10.1 |

> **实务口诀**：管理评审输入 **"上变七（成）供机改资"**——上次跟进 / 内外部变化 / 顾客满意 / 目标达成 / 过程绩效 / 不合格纠正 / 监视测量 / 审核结果 / 外部供方 / 风险措施有效性 / 资源充分性 / 改进机会。

> **审核员追问**："少一项 = 不符合 9.3.2"。

---

### 9.3.3 输出（**3 件·必背**）

> 管理评审的输出应包括：

| # | 输出项 | 含义 |
|---|---|---|
| 1 | **改进的机会** | 哪些方面能改得更好 |
| 2 | **QMS 所需的变更** | 体系本身要不要调整 |
| 3 | **资源需求** | 需要增加什么资源 |

> ⚠️ **少一项 = 不符合 9.3.3**

> **保留成文信息**：会议纪要 + 决议清单 + 跟进表（**9.3.3 最后一句原文**）

## 六、9.3 管理评审的实务流程

```
年度管理评审会议
├── 主持：总经理（必须——委托副总代开可能不被认可）
├── 出席：副总（生产/技术/质量/销售）、各部门经理
├── 缺席：财务总监（指定书面输入）
│
├── 输入材料（按 9.3.2 顺序准备 12 项）
│   ├── 1. 上次决议跟进（9 项，已闭环 8 项，1 项延期）
│   ├── 2. 内外部变化：原材料涨价 15%；欧盟 RoHS 新指令
│   ├── 3. QMS 绩效数据（详见下）
│   ├── ...
│
├── 输出决定（按 9.3.3 三件）
│   ├── 1. 改进机会：投资 200 万升级自动化测试线
│   ├── 2. QMS 变更：新增 RoHS 合规检查流程
│   └── 3. 资源需求：批准质检员招聘 + 培训预算 30 万
│
└── 跟进：季度回顾 + 下年度管理评审
```

### QMS 绩效数据示例（输入第 3 项）

| 指标 | 实际 | 目标 | 判定 |
|---|---|---|---|
| 顾客满意度 | 89 分 | 90 分 | 差 1 分 |
| 一次合格率 | 97.5% | 97% | 达标 |
| 准时交付率 | 94% | 95% | 差 1% |
| 内审不符合 | 12 项 | — | 关闭 10 项 |
| 外审 | 0 重大不符合 | — | 达标 |
| 供方绩效 | A 5 / B 3 / C 1 淘汰 | — | 持续优化 |

## 八、文控岗在审核中的"3 大任务"

1. **陪同**：全程陪同外审员，回答文件相关问题
2. **记录**：实时记录审核员的提问和发现
3. **应对**：现场抽文件时 30 秒内调出（靠文件总清单 + 借阅登记）

---

## 九、面试杀手锏：被问"9.3 管理评审"怎么答

> 答：
> 管理评审在 9.3 章，是最高管理者按策划时间间隔对 QMS 的全面体检，目的是**确保 QMS 的适宜性、充分性、有效性，并与战略方向一致**。
> **输入有 12 项**（按 9.3.2 a-e 分类）：上次跟进、内外部变化、QMS 绩效七大件（顾客满意/目标/过程/不合格/监视/审核/供方/风险措施有效性）、资源充分性、改进机会。
> **输出必须 3 件**：改进机会 + QMS 变更 + 资源需求。少一件 = 不符合。
> 文控岗在管理评审中的工作是**归档会议纪要、决议清单、跟进表**（9.3.3 最后一句原文要求保留成文信息）。

## 十、自测题

1. 9.2 内部审核的 3 个目的是什么？
2. 9.2.2 审核方案包括哪些要素（9 要素）？
3. 9.3 管理评审的 3 件输出是什么？

5. 审核员不能审什么？

**参考答案**：

**1.** 9.2.1：a) 符合组织自身要求；b) 符合本标准要求；c) 得到有效实施和保持
